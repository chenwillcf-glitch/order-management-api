# Development Guide\n\nThis guide provides information for developers working on extending and improving the Order Management API.\n\n## Architecture Overview\n\nThe application follows a layered architecture:\n\n```\nController Layer (REST endpoints)\n        ↓\nService Layer (Business logic)\n        ↓\nRepository Layer (Data access)\n        ↓\nEntity Models (Domain objects)\n        ↓\nDatabase (H2 in-memory)\n```\n\n## Code Structure\n\n### Controllers (`com.orderapi.controller`)\n- Handle HTTP requests and responses\n- Validate input from clients\n- Call appropriate service methods\n- Return responses with proper HTTP status codes\n\n**Files:**\n- `OrderController.java` - Order endpoints\n- `ProductController.java` - Product endpoints\n- `CustomerController.java` - Customer endpoints\n\n### Services (`com.orderapi.service`)\n- Implement business logic\n- Coordinate between repositories\n- Perform data validation and transformation\n- Handle transactions\n\n**Files:**\n- `OrderService.java` - Order business logic\n- `ProductService.java` - Product business logic\n- `CustomerService.java` - Customer business logic\n\n### Repositories (`com.orderapi.repository`)\n- Data access layer using Spring Data JPA\n- Define query methods\n- Handle database operations\n\n**Files:**\n- `OrderRepository.java`\n- `ProductRepository.java`\n- `CustomerRepository.java`\n\n### Models (`com.orderapi.model`)\n- JPA entity classes\n- Represent domain objects\n- Define database schema through annotations\n\n**Files:**\n- `Order.java` - Order entity\n- `Product.java` - Product entity\n- `Customer.java` - Customer entity\n\n## Database Schema\n\n### Products Table\n```sql\nCREATE TABLE products (\n    id BIGINT PRIMARY KEY AUTO_INCREMENT,\n    name VARCHAR(255) NOT NULL,\n    description TEXT,\n    price DECIMAL(10, 2) NOT NULL,\n    quantity INT,\n    sku VARCHAR(255)\n);\n```\n\n### Customers Table\n```sql\nCREATE TABLE customers (\n    id BIGINT PRIMARY KEY AUTO_INCREMENT,\n    first_name VARCHAR(255) NOT NULL,\n    last_name VARCHAR(255) NOT NULL,\n    email VARCHAR(255),\n    phone_number VARCHAR(255),\n    address VARCHAR(255) NOT NULL,\n    city VARCHAR(255),\n    state VARCHAR(255),\n    zip_code VARCHAR(255)\n);\n```\n\n### Orders Table\n```sql\nCREATE TABLE orders (\n    id BIGINT PRIMARY KEY AUTO_INCREMENT,\n    customer_id BIGINT,\n    total_amount DECIMAL(10, 2) NOT NULL,\n    status VARCHAR(255) NOT NULL,\n    created_at TIMESTAMP,\n    updated_at TIMESTAMP,\n    notes TEXT,\n    FOREIGN KEY (customer_id) REFERENCES customers(id)\n);\n```\n\n### Order Items Join Table\n```sql\nCREATE TABLE order_items (\n    order_id BIGINT,\n    product_id BIGINT,\n    PRIMARY KEY (order_id, product_id),\n    FOREIGN KEY (order_id) REFERENCES orders(id),\n    FOREIGN KEY (product_id) REFERENCES products(id)\n);\n```\n\n## Adding a New Feature\n\n### 1. Create Entity Model\n- Add new class in `com.orderapi.model`\n- Annotate with `@Entity` and `@Table`\n- Define fields with appropriate JPA annotations\n- Add constructors, getters, and setters\n\n### 2. Create Repository\n- Add new interface in `com.orderapi.repository`\n- Extend `JpaRepository<Entity, ID>`\n- Define custom query methods if needed\n\n### 3. Create Service\n- Add new class in `com.orderapi.service`\n- Annotate with `@Service`\n- Inject repository using `@Autowired`\n- Implement business logic methods\n- Add validation and error handling\n\n### 4. Create Controller\n- Add new class in `com.orderapi.controller`\n- Annotate with `@RestController` and `@RequestMapping`\n- Inject service using `@Autowired`\n- Define endpoints with proper HTTP methods\n- Handle request/response mapping\n\n### 5. Add Tests\n- Create test class in `src/test/java`\n- Add unit tests for service layer\n- Add integration tests for controller layer\n- Test edge cases and error scenarios\n\n## Best Practices\n\n### Error Handling\n- Use custom exceptions for domain-specific errors\n- Implement global exception handler with `@ControllerAdvice`\n- Return appropriate HTTP status codes\n- Provide meaningful error messages\n\n### Validation\n- Use JSR-303 annotations (`@NotNull`, `@Min`, `@Email`, etc.)\n- Add custom validators for complex validation\n- Validate at controller and service layers\n- Return 400 Bad Request for validation failures\n\n### Transactions\n- Use `@Transactional` for operations spanning multiple entities\n- Set `readOnly = true` for query-only methods\n- Handle rollback scenarios\n- Use appropriate isolation levels\n\n### Logging\n- Use SLF4J with appropriate log levels\n- Log at INFO level for business operations\n- Log at DEBUG level for detailed information\n- Log at ERROR level for exceptions\n- Include relevant context in log messages\n\n### Testing\n- Write unit tests for services\n- Write integration tests for controllers\n- Use mocks for dependencies\n- Test both happy path and error scenarios\n- Aim for >80% code coverage\n\n## Running the Application\n\n### Build\n```bash\nmvn clean install\n```\n\n### Run\n```bash\nmvn spring-boot:run\n```\n\n### Test\n```bash\nmvn test\n```\n\n### Test with Coverage\n```bash\nmvn test jacoco:report\n```\n\n## Configuration\n\nApplication properties are defined in `src/main/resources/application.properties`:\n\n- `spring.application.name` - Application name\n- `server.port` - Server port (default: 8080)\n- `spring.datasource.url` - Database URL\n- `spring.jpa.hibernate.ddl-auto` - DDL generation strategy\n- `spring.h2.console.enabled` - H2 console access\n\n## Debugging\n\n### H2 Console\nAccess the H2 database console at: `http://localhost:8080/h2-console`\n\n### Application Logs\nCheck console output for application logs and errors.\n\n### IDE Debugging\n- Set breakpoints in your code\n- Run application in debug mode\n- Use debugger to step through code\n\n## Common Tasks\n\n### Add a New Endpoint\n1. Add method to controller class\n2. Add `@RequestMapping` or specific HTTP method annotation\n3. Define path and request/response types\n4. Call service method\n5. Return appropriate response\n\n### Modify Database Schema\n1. Update entity model\n2. Hibernate will auto-generate schema (if ddl-auto is set to `update` or `create`)\n3. For production, use database migration tools (Flyway, Liquibase)\n\n### Add Custom Query\n1. Define method signature in repository interface\n2. Annotate with `@Query` if needed\n3. Implement in service using repository method\n4. Add tests for new query\n\n## Resources\n\n- [Spring Boot Documentation](https://spring.io/projects/spring-boot)\n- [Spring Data JPA](https://spring.io/projects/spring-data-jpa)\n- [Spring Web](https://spring.io/guides/gs/serving-web-content/)\n- [JUnit 5](https://junit.org/junit5/)\n- [Mockito](https://site.mockito.org/)\n
